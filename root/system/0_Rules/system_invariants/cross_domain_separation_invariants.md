# Cross‑Domain Separation Invariants

Cross‑domain separation invariants define the highest‑level membranes that prevent fusion, inversion, contamination, or reinterpretation between domains.

These invariants are upstream of all domain physics and cannot be overridden.

---

# 1. Domain Identity Separation

## 1.1 Each domain has a fixed identity
Domains must remain:
- distinct  
- isolated  
- non‑porous  
- non‑interchangeable  

## 1.2 No domain may absorb another
Absorption includes:
- taking on another domain’s role  
- inheriting another domain’s properties  
- reinterpreting another domain’s boundaries  

## 1.3 No domain may rename or reinterpret another
Naming or interpreting another domain is a violation of separation.

---

# 2. Domain Function Separation

## 2.1 Each domain has a single function
Domains must not:
- expand their function  
- reinterpret their function  
- take on additional functions  

## 2.2 No domain may perform another domain’s function
Examples of violations:
- structure performing execution  
- engine performing interpretation  
- registry performing naming  
- semantics performing routing  

## 2.3 No domain may delegate its function to another
Delegation is fusion.

---

# 3. Domain Boundary Separation

## 3.1 Boundaries between domains must remain intact
Boundaries cannot:
- soften  
- blur  
- reinterpret  
- collapse  

## 3.2 Boundaries cannot be crossed by semantics
Meaning cannot:
- alter boundaries  
- reinterpret boundaries  
- weaken boundaries  

## 3.3 Boundaries cannot be crossed by execution
Execution cannot:
- mutate boundaries  
- bypass boundaries  
- reinterpret boundaries  

---

# 4. Domain Influence Separation

## 4.1 Upstream domains cannot be influenced by downstream domains
Downstream domains cannot change:
- invariants  
- geometry  
- structure  
- naming rules  
- categories  

## 4.2 Semantics cannot influence upstream physics
Semantics cannot:
- alter geometry  
- alter structure  
- alter naming  
- alter categories  

## 4.3 Execution cannot influence upstream physics
Execution cannot:
- mutate geometry  
- mutate boundaries  
- mutate categories  

---

# 5. Domain Interpretation Separation

## 5.1 No domain may interpret another domain
Interpretation includes:
- assigning meaning  
- inferring intent  
- reinterpreting structure  
- reinterpreting geometry  

## 5.2 No domain may reinterpret itself
Self‑reinterpretation is drift.

## 5.3 No domain may reinterpret naming
Naming is structural, not semantic.

---

# 6. Domain Routing Separation

## 6.1 Routing cannot cross domain boundaries
Routing cannot:
- reinterpret structure  
- reinterpret geometry  
- reinterpret categories  

## 6.2 Routing cannot create cross‑domain pathways
Routing cannot:
- invent new adjacency  
- invent new placement  
- invent new transitions  

## 6.3 Routing cannot override domain separation
Routing must respect all membranes.

---

# 7. Domain Containment Separation

## 7.1 Each domain must remain contained within its own membrane
Containment prevents:
- fusion  
- inversion  
- contamination  
- reinterpretation  

## 7.2 No domain may leak into another
Leakage includes:
- meaning  
- identity  
- structure  
- geometry  
- categories  
- routing  

## 7.3 Containment is upstream of collapse
Containment failure increases collapse risk.

---

# 8. Domain Temporal Separation

## 8.1 Domains cannot drift over time
Domains must remain:
- stable  
- fixed  
- non‑evolving  

## 8.2 Execution cannot cause temporal drift
Execution cannot:
- accumulate residue  
- alter domain identity  
- reinterpret domain boundaries  

---

# 9. Domain Hierarchy Separation

## 9.1 No domain may become hierarchical over another
Domains cannot:
- command  
- dominate  
- override  
- reinterpret  

## 9.2 No domain may become subordinate to another
Subordination is fusion.

## 9.3 Coordination is relational, not hierarchical
Coordination cannot create hierarchy.

---

# 10. Domain Semantic Separation

## 10.1 Semantics cannot cross domain boundaries
Semantics cannot:
- influence structure  
- influence geometry  
- influence naming  
- influence categories  

## 10.2 Semantics cannot reinterpret other domains
Meaning cannot:
- redefine shapes  
- redefine boundaries  
- redefine categories  

## 10.3 Semantics cannot fuse with content
Semantics ≠ content.
