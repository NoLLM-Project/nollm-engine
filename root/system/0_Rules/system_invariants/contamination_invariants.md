# Contamination Invariants

Contamination invariants prevent semantic, structural, or categorical bleed across domains.  
They ensure that boundaries remain intact and that no domain absorbs or reinterprets another.

These invariants are upstream of execution and non‑negotiable.

---

# 1. Domain Contamination Invariant

## 1.1 No domain may absorb another
Domains must remain:
- separate  
- isolated  
- non‑porous  

No domain may fuse with:
- structure  
- engine  
- registry  
- semantics  
- geometry  
- placement  
- adjacency  

## 1.2 No domain may reinterpret another
Domains cannot:
- assign meaning to another domain  
- reinterpret another domain’s shapes  
- reinterpret another domain’s boundaries  

Interpretation is contamination.

---

# 2. Semantic Contamination Invariant

## 2.1 Semantics cannot bleed into structure
Semantics cannot:
- redefine shapes  
- redefine boundaries  
- redefine geometry  

## 2.2 Semantics cannot bleed into naming
Naming cannot become meaning.  
Meaning cannot become naming.

## 2.3 Semantics cannot bleed into execution
Execution is mechanical, not semantic.

---

# 3. Structural Contamination Invariant

## 3.1 Structure cannot absorb semantics
Structure cannot:
- encode meaning  
- encode identity  
- encode interpretation  

## 3.2 Structure cannot absorb registry
Structure cannot:
- store registry data  
- store references  
- store metadata  

## 3.3 Structure cannot absorb engine logic
Structure cannot:
- execute  
- route  
- interpret  

---

# 4. Registry Contamination Invariant

## 4.1 Registry cannot absorb semantics
Registry entries cannot:
- define meaning  
- define identity  
- define categories  

## 4.2 Registry cannot absorb execution
Registry cannot:
- run  
- interpret  
- route  

## 4.3 Registry cannot absorb structure
Registry cannot:
- define shapes  
- define geometry  
- define boundaries  

---

# 5. Naming Contamination Invariant

## 5.1 Naming cannot absorb semantics
Names cannot:
- encode meaning  
- encode identity  
- encode interpretation  

## 5.2 Naming cannot absorb structure
Names cannot:
- define shapes  
- define geometry  
- define boundaries  

## 5.3 Naming cannot absorb categories
Names cannot create or redefine categories.

---

# 6. Geometry Contamination Invariant

## 6.1 Geometry cannot absorb semantics
Geometry cannot:
- encode meaning  
- encode identity  
- encode interpretation  

## 6.2 Geometry cannot absorb placement
Geometry defines form; placement uses form.

## 6.3 Geometry cannot absorb adjacency
Geometry defines shape; adjacency defines contact.

---

# 7. Placement Contamination Invariant

## 7.1 Placement cannot absorb semantics
Placement cannot:
- interpret meaning  
- assign identity  
- infer intent  

## 7.2 Placement cannot absorb geometry
Placement uses geometry; it does not define it.

## 7.3 Placement cannot absorb routing
Placement is static; routing is dynamic.

---

# 8. Adjacency Contamination Invariant

## 8.1 Adjacency cannot absorb semantics
Adjacency is structural, not semantic.

## 8.2 Adjacency cannot absorb geometry
Adjacency uses geometry; it does not define it.

## 8.3 Adjacency cannot absorb placement
Adjacency defines contact; placement defines location.

---

# 9. Execution Contamination Invariant

## 9.1 Execution cannot absorb semantics
Execution cannot:
- interpret meaning  
- infer intent  
- assign semantics  

## 9.2 Execution cannot absorb registry
Execution cannot:
- store data  
- mutate registry  
- reinterpret registry entries  

## 9.3 Execution cannot absorb structure
Execution cannot:
- mutate shapes  
- mutate boundaries  
- mutate geometry  

---

# 10. Cross‑Domain Contamination Invariant

## 10.1 No domain may leak into another
Leakage includes:
- meaning  
- identity  
- structure  
- geometry  
- categories  
- routing  
- placement  

## 10.2 All contamination is a violation
Any cross‑domain bleed is a structural failure.

## 10.3 Contamination is upstream of collapse
Contamination triggers:
- drift  
- distortion  
- collapse risk  

Contamination must be detected early and isolated immediately.
