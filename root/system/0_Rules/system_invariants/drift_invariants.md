# Drift Invariants

Drift invariants define the non‑negotiable physics that prevent slow, silent, or cumulative deformation of system structure, boundaries, or identity.

Drift is always a violation.  
Drift is always upstream of collapse.  
Drift must always be detected early.

---

# 1. Identity Drift Invariant

## 1.1 Identity cannot drift
Identity must remain:
- fixed  
- stable  
- non‑porous  
- non‑evolving  

Identity cannot:
- reinterpret itself  
- rename itself  
- split  
- fuse  

## 1.2 Identity cannot be influenced by downstream domains
Identity cannot be changed by:
- execution  
- semantics  
- registry  
- engine  
- naming  

Identity is upstream of all domains.

---

# 2. Boundary Drift Invariant

## 2.1 Boundaries cannot drift
Boundaries must remain:
- explicit  
- intact  
- non‑porous  

Boundaries cannot:
- soften  
- reinterpret  
- expand  
- contract  

## 2.2 Boundaries cannot be influenced by semantics
Meaning cannot:
- alter boundaries  
- reinterpret boundaries  
- weaken boundaries  

Boundaries are structural, not semantic.

---

# 3. Category Drift Invariant

## 3.1 Categories cannot drift
Categories must remain:
- fixed  
- non‑evolving  
- non‑semantic  

Categories cannot:
- reinterpret themselves  
- absorb new members  
- lose members  
- fuse with other categories  

## 3.2 Categories cannot be created by downstream domains
Execution, semantics, or registry cannot create new categories.

---

# 4. Naming Drift Invariant

## 4.1 Names cannot drift
Names must remain:
- stable  
- structural  
- non‑semantic  

Names cannot:
- reinterpret themselves  
- accumulate meaning  
- shed meaning  
- shift identity  

## 4.2 Naming cannot become meaning
Naming → structure, never reversed.

---

# 11. Cross‑Domain Drift Invariant

## 11.1 No domain may drift into another
Drift includes:
- fusion  
- inversion  
- contamination  
- reinterpretation  

## 11.2 Drift must remain local
Drift cannot propagate across domains.

## 11.3 Drift is upstream of collapse
Any drift increases collapse risk and must be detected early.

---

# 12. Temporal Drift Invariant

## 12.1 System physics cannot drift over time
The following must remain stable across time:
- identity  
- boundaries  
- categories  
- geometry  
- naming rules  

## 12.2 Execution cannot cause temporal drift
Execution cannot:
- accumulate residue  
- alter physics  
- reinterpret upstream domains  

---

# 13. Drift Detection Invariant

## 13.1 Drift must be detectable
The system must detect:
- boundary drift  
- category drift  
- naming drift  
- adjacency drift  
- placement drift  
- semantic drift  

## 13.2 Drift detection is structural, not semantic
Detection cannot:
- interpret meaning  
- infer intent  
- assign semantics  

## 13.3 Drift detection must be upstream of execution
Drift must be detected before the engine runs.
