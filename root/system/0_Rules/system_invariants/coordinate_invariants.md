# Coordinate Invariants

Coordinate invariants define the non‑negotiable physics of coordinates as structural entities.  
Coordinates are upstream of placement, adjacency, routing, and execution.

Coordinates cannot be interpreted, renamed, repurposed, or mutated.

---

# 1. Coordinate Identity Invariant

## 1.1 Coordinates are structural, not semantic
Coordinates cannot:
- encode meaning  
- encode identity  
- encode interpretation  

Coordinates are pure structure.

## 1.2 Coordinates cannot be renamed
Names cannot:
- redefine coordinates  
- reinterpret coordinates  
- assign semantics to coordinates  

Coordinate identity is fixed.

---

# 2. Coordinate Stability Invariant

## 2.1 Coordinates cannot drift
Coordinates cannot:
- slide  
- reinterpret themselves  
- accumulate residue  
- change dimensionality  

## 2.2 Coordinates cannot be mutated by execution
Execution cannot:
- create coordinates  
- destroy coordinates  
- distort coordinates  

Coordinates are upstream of engine movement.

---

# 3. Coordinate Boundary Invariant

## 3.1 Coordinates must respect geometry
Coordinates must obey:
- shapes  
- boundaries  
- dimensions  

Coordinates cannot violate geometry.

## 3.2 Coordinates cannot cross domain boundaries
Coordinates cannot:
- enter registry  
- enter semantics  
- enter engine logic  

Coordinates remain in the structural domain.

---

# 4. Coordinate Placement Invariant

## 4.1 Placement uses coordinates; it does not define them
Placement cannot:
- create coordinates  
- mutate coordinates  
- reinterpret coordinates  

## 4.2 Coordinates cannot infer adjacency
Adjacency is structural; coordinates cannot create or imply adjacency.

---

# 5. Coordinate Routing Invariant

## 5.1 Routing cannot reinterpret coordinates
Routing cannot:
- rename coordinates  
- repurpose coordinates  
- infer meaning from coordinates  

## 5.2 Routing cannot create new coordinate transitions
Routing must use only:
- existing adjacency  
- existing placement  

No new transitions may be invented.

---

# 6. Coordinate Contamination Invariant

## 6.1 Coordinates cannot absorb semantics
Coordinates cannot:
- encode meaning  
- encode categories  
- encode identity  

## 6.2 Coordinates cannot absorb execution
Coordinates cannot:
- store state  
- store results  
- store metadata  

Coordinates remain pure structure.
