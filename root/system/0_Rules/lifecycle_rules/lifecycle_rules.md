# Lifecycle Rules

## 1. Lifecycle is structural, not semantic
Lifecycle defines:
- allowed phases  
- allowed transitions between phases  

Lifecycle does not define:
- meaning  
- category  
- semantics  
- interpretation  

Lifecycle is non‑semantic.

## 2. Lifecycle is downstream of identity and state
Identity defines:
- what persists  

State defines:
- what varies  

Lifecycle defines:
- how existence is phased  

Identity → State → Lifecycle.

## 3. Lifecycle cannot assign identity
Lifecycle does not:
- define what something is  
- define what something means  
- define category  
- define semantics  

Lifecycle is not identity.

## 4. Lifecycle is constrained by invariants
Lifecycle must obey:
- system invariants  
- non‑mutation  
- non‑fusion  
- non‑inversion  

Lifecycle cannot violate invariants.

## 5. Lifecycle prevents semantic drift
Lifecycle ensures:
- phases do not imply meaning  
- phases do not imply category  
- phases do not imply identity  

Lifecycle is a membrane.

## 6. Lifecycle transitions are reversible
Lifecycle transitions:
- do not mutate identity  
- do not mutate semantics  
- do not mutate structure  

Lifecycle leaves no residue.

## 7. Lifecycle cannot create new domains
Lifecycle cannot:
- create new categories  
- create new semantics  
- create new containers  
- create new routing paths  

Lifecycle only phases what exists.

## 8. Lifecycle must be explicit
Lifecycle must be:
- declared  
- bounded  
- intentional  

Implicit lifecycle leads to drift.

## 9. Lifecycle cannot interpret behavior
Lifecycle does not:
- infer intent  
- interpret meaning  
- evaluate content  

Lifecycle is structural, not cognitive.

## 10. Lifecycle must remain isolated
Lifecycle must not fuse with:
- identity  
- semantics  
- registry  
- placement  
- adjacency  
- routing  
- engine logic  

Lifecycle is a standalone conceptual domain.
