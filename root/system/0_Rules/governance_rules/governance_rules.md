# Governance Rules

## 1. Governance protects invariants, not behavior
Governance ensures:
- invariants remain intact  
- boundaries remain intact  
- categories remain intact  

Governance does not:
- interpret meaning  
- enforce semantics  
- control behavior  

Governance protects the system’s physics, not its content.

## 2. Governance is upstream of all domains
Governance sits above:
- structure  
- engine  
- registry  
- semantics  
- geometry  
- placement  
- adjacency  

Governance does not depend on these domains.

## 3. Governance is non‑interpretive
Governance does not:
- interpret meaning  
- infer intent  
- assign semantics  
- evaluate content  

Governance is structural oversight, not semantic oversight.

## 4. Governance prevents domain fusion
Governance ensures:
- structure ≠ engine  
- engine ≠ registry  
- registry ≠ semantics  
- semantics ≠ content  
- geometry ≠ placement  

Governance maintains separation between domains.

## 5. Governance prevents inversion
Governance ensures:
- engine never becomes structure  
- structure never becomes semantics  
- naming never becomes meaning  
- registry never becomes execution  

Governance protects directionality.

## 6. Governance is non‑mutative
Governance cannot:
- change rules  
- change categories  
- change boundaries  
- change geometry  
- change naming rules  

Governance preserves; it does not mutate.

## 7. Governance is reversible
Governance actions:
- leave no residue  
- do not mutate identity  
- do not mutate semantics  
- do not mutate structure  

Governance is oversight, not transformation.

## 8. Governance cannot create new domains
Governance cannot:
- create new categories  
- create new semantics  
- create new containers  
- create new routing paths  

Governance only protects what already exists.

## 9. Governance is explicit, not implicit
Governance must be:
- declared  
- bounded  
- intentional  

Implicit governance leads to drift.

## 10. Governance prevents systemic drift
Governance ensures:
- invariants remain stable  
- membranes remain intact  
- boundaries remain clear  
- domains remain separate  

Governance is the system’s long‑term stabilizer.
