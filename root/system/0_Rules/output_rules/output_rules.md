# Output Rules

## 1. Output is structural, not semantic
Output defines:
- how results are represented  
- how structure is emitted  

Output does not define:
- meaning  
- category  
- semantics  
- interpretation  

Output is non‑semantic.

## 2. Output is downstream of the engine
The engine defines:
- mechanical processing  

Output defines:
- structural representation of results  

Output cannot override engine behavior.

## 3. Output cannot assign identity
Output does not:
- define what something is  
- define what something means  
- define category  
- define semantics  

Output is not identity.

## 4. Output is constrained by invariants
Output must obey:
- system invariants  
- non‑mutation  
- non‑fusion  
- non‑inversion  

Output cannot violate invariants.

## 5. Output prevents semantic drift
Output ensures:
- representation does not imply meaning  
- representation does not imply category  
- representation does not imply identity  

Output is a membrane.

## 6. Output is reversible
Output operations:
- do not mutate identity  
- do not mutate semantics  
- do not mutate structure  

Output leaves no residue.

## 7. Output cannot create new domains
Output cannot:
- create new categories  
- create new semantics  
- create new containers  
- create new routing paths  

Output only represents what exists.

## 8. Output must be explicit
Output must be:
- declared  
- bounded  
- intentional  

Implicit output leads to drift.

## 9. Output cannot interpret behavior
Output does not:
- infer intent  
- interpret meaning  
- evaluate content  

Output is structural, not cognitive.

## 10. Output must remain isolated
Output must not fuse with:
- semantics  
- identity  
- registry  
- placement  
- adjacency  
- routing  
- lifecycle  
- state  
- engine logic  

Output is a standalone conceptual domain.
