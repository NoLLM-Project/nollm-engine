# Semantic Rules

The semantic layer defines meaning, interpretation, conceptual relationships, and human‑facing significance.  
Semantics is the first domain above content and the first domain allowed to carry meaning.

Semantics is interpretive.  
Semantics is conceptual.  
Semantics is descriptive.

Semantics is not structural.  
Semantics is not operational.  
Semantics is not generative of upstream physics.

Semantics interprets; it does not construct.

---

# 1. Semantic Definition Rules

## 1.1 Semantics defines meaning and interpretation
Semantics may contain:
- conceptual meaning  
- interpretive relationships  
- descriptive metadata  
- human‑facing significance  
- semantic groupings  
- semantic relationships  

Semantics is the first layer allowed to interpret.

## 1.2 Semantics cannot contain structural information
Forbidden semantic contents:
- geometry  
- adjacency  
- placement  
- invariants  
- engine logic  
- routing logic  
- workflow structure  
- pipeline structure  
- execution logic  
- content structure  

Semantics cannot encode physics.

---

# 2. Semantic Position Rules

## 2.1 Semantics is downstream of content
Semantics may:
- interpret content  
- assign meaning to content  
- group content semantically  

Semantics may not mutate content.

## 2.2 Semantics is downstream of all structural domains
Semantics may not:
- reinterpret geometry  
- reinterpret adjacency  
- reinterpret placement  
- reinterpret naming rules  
- reinterpret categories  
- reinterpret engine behavior  
- reinterpret routing transitions  
- reinterpret workflow steps  
- reinterpret pipeline stages  
- reinterpret execution operations  

Semantics cannot reinterpret structure.

## 2.3 Semantics is upstream of user‑facing layers
Semantics provides meaning to:
- UI  
- narrative  
- human‑facing systems  
- interpretation layers  

Semantics does not perform operations.

---

# 3. Semantic Visibility Rules

## 3.1 Semantics may see content surfaces
Allowed visibility:
- content states  
- content transitions  
- content boundaries  
- content material  

## 3.2 Semantics may not see upstream structural domains
Forbidden visibility:
- pipeline structure  
- workflow structure  
- routing tables  
- geometry definitions  
- naming rules  
- invariants  
- registry internals  

Semantics cannot reveal upstream physics.

---

# 4. Semantic Interpretation Rules

## 4.1 Semantics may interpret meaning
Semantics may:
- assign meaning  
- define conceptual relationships  
- define semantic groupings  
- define interpretive metadata  

## 4.2 Semantics may not reinterpret structure
Semantics may not:
- reinterpret transitions  
- reinterpret boundaries  
- reinterpret adjacency  
- reinterpret placement  
- reinterpret geometry  

Semantics interprets content, not structure.

---

# 5. Semantic Mutation Rules

## 5.1 Semantics cannot mutate upstream domains
Semantics cannot change:
- content  
- execution  
- pipelines  
- workflows  
- routing  
- geometry  
- adjacency  
- placement  
- naming  
- categories  
- invariants  

Semantics is downstream.

## 5.2 Semantics cannot generate new structural domains
Semantics cannot create:
- new content types  
- new execution types  
- new pipeline types  
- new workflow types  
- new routing types  
- new adjacency types  
- new invariants  

Semantics is not structural.

---

# 6. Semantic Boundary Rules

## 6.1 Semantics cannot cross into structural domains
Semantics cannot enter:
- engine logic  
- routing logic  
- workflow logic  
- pipeline logic  
- execution logic  
- content logic  

Semantics remains interpretive.

## 6.2 Semantics cannot fuse with structural domains
Semantics cannot become:
- content  
- execution  
- pipeline  
- workflow  
- routing  
- engine  
- geometry  

Fusion is contamination.

---

# 7. Semantic Stability Rules

## 7.1 Semantics must be deterministic
Given the same content:
- semantic meaning must be identical  
- semantic relationships must be identical  
- semantic interpretation must be identical  

## 7.2 Semantics cannot drift
Semantics cannot:
- reinterpret itself over time  
- accumulate meaning  
- adapt based on usage  
- evolve without explicit definition  

Semantics is stable.
