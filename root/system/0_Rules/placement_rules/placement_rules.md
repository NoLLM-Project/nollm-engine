# Placement Rules

## 1. Placement is structural, not semantic
Placement defines:
- where something can be  
- where something cannot be  

Placement does not define:
- what something means  
- what something is  
- how something behaves  

Placement is non‑semantic.

## 2. Placement is upstream of routing
Placement establishes:
- allowed positions  
- allowed relationships  

Routing determines:
- movement between positions  

Placement comes first.

## 3. Placement cannot assign identity
Placement does not:
- define identity  
- define category  
- define meaning  
- define semantics  

Placement is not naming.

## 4. Placement is constrained by geometry
Placement must obey:
- shape boundaries  
- dimensional constraints  
- geometric forms  

Placement cannot violate geometry.

## 5. Placement prevents spatial contamination
Placement ensures:
- objects do not overlap improperly  
- domains do not fuse  
- boundaries remain intact  

Placement is a membrane.

## 6. Placement is reversible
Placement actions:
- do not mutate identity  
- do not mutate semantics  
- do not mutate structure  

Placement leaves no residue.

## 7. Placement cannot create new domains
Placement cannot:
- create new categories  
- create new semantics  
- create new containers  
- create new routing paths  

Placement only uses existing structure.

## 8. Placement is explicit, not implicit
Placement must be:
- declared  
- bounded  
- intentional  

Implicit placement leads to drift.

## 9. Placement prevents adjacency inversion
Placement ensures:
- adjacency remains adjacency  
- adjacency does not become meaning  
- adjacency does not become identity  

Placement protects adjacency from semantic drift.

## 10. Placement must remain isolated
Placement must not fuse with:
- geometry  
- adjacency  
- routing  
- semantics  
- registry  
- engine logic  

Placement is a standalone conceptual domain.
