# Engine Rules

## 1. Engine is mechanical, not semantic
The engine defines:
- how structure is processed  
- how transitions occur mechanically  

The engine does not define:
- meaning  
- category  
- semantics  
- interpretation  

The engine is non‑semantic.

## 2. Engine is downstream of all structural domains
The engine depends on:
- identity  
- state  
- lifecycle  
- registry  
- routing  
- placement  
- adjacency  
- geometry  
- invariants  

The engine cannot override any upstream domain.

## 3. Engine cannot assign identity
The engine does not:
- define what something is  
- define what something means  
- define category  
- define semantics  

The engine is not identity.

## 4. Engine is constrained by invariants
The engine must obey:
- system invariants  
- non‑mutation  
- non‑fusion  
- non‑inversion  

The engine cannot violate invariants.

## 5. Engine prevents semantic drift
The engine ensures:
- processing does not imply meaning  
- transitions do not imply category  
- outputs do not imply identity  

The engine is a membrane.

## 6. Engine operations are reversible
Engine operations:
- do not mutate identity  
- do not mutate semantics  
- do not mutate structure  

The engine leaves no residue.

## 7. Engine cannot create new domains
The engine cannot:
- create new categories  
- create new semantics  
- create new containers  
- create new routing paths  

The engine only processes what exists.

## 8. Engine must be explicit
Engine behavior must be:
- declared  
- bounded  
- intentional  

Implicit engine behavior leads to drift.

## 9. Engine cannot interpret behavior
The engine does not:
- infer intent  
- interpret meaning  
- evaluate content semantically  

The engine is mechanical, not cognitive.

## 10. Engine must remain isolated
The engine must not fuse with:
- semantics  
- identity  
- registry  
- placement  
- adjacency  
- routing  
- lifecycle  
- state  

The engine is a standalone conceptual domain.
