# Invariants Rules

## 1. Invariants define system physics, not behavior
Invariants describe:
- constraints  
- boundaries  
- membranes  
- non‑negotiables  

Invariants do not describe:
- behavior  
- semantics  
- content  
- execution  

Invariants define the physics of the system.

## 2. Invariants are upstream of all domains
Invariants sit above:
- structure  
- engine  
- registry  
- semantics  
- geometry  
- governance  

Invariants do not depend on these domains.

## 3. Invariants are non‑mutative
Invariants cannot:
- change  
- evolve  
- adapt  
- reinterpret themselves  
- be modified by execution  

Invariants are fixed and stable.

## 4. Invariants are non‑interpretive
Invariants do not:
- interpret meaning  
- infer intent  
- assign semantics  

Invariants are structural truths, not semantic constructs.

## 5. Invariants prevent fusion
Invariants ensure:
- structure ≠ engine  
- engine ≠ registry  
- registry ≠ semantics  
- semantics ≠ content  
- geometry ≠ placement  

Invariants maintain separation between domains.

## 6. Invariants prevent inversion
Invariants ensure:
- naming never becomes meaning  
- structure never becomes semantics  
- engine never becomes structure  
- registry never becomes execution  

Invariants protect directionality.

## 7. Invariants are reversible
Invariant enforcement:
- leaves no residue  
- does not mutate identity  
- does not mutate semantics  
- does not mutate structure  

Invariants enforce without altering.

## 8. Invariants cannot create new domains
Invariants cannot:
- create new categories  
- create new semantics  
- create new containers  
- create new routing paths  

Invariants only constrain what already exists.

## 9. Invariants must be explicit
Invariants must be:
- declared  
- bounded  
- intentional  

Implicit invariants lead to drift.

## 10. Invariants remain stable across time
Invariants ensure:
- system identity remains stable  
- boundaries remain stable  
- membranes remain stable  
- domains remain separate  

Invariants are the long‑term anchor of the system.
