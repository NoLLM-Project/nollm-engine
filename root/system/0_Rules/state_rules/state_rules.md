# State Rules

## 1. State is structural, not semantic
State defines:
- what can change  
- what cannot change  
- how change is represented  

State does not define:
- meaning  
- category  
- semantics  
- interpretation  

State is non‑semantic.

## 2. State is downstream of identity
Identity defines:
- what persists  

State defines:
- what varies  

Identity comes first.

## 3. State cannot assign identity
State does not:
- define what something is  
- define what something means  
- define category  
- define semantics  

State is not identity.

## 4. State is constrained by invariants
State must obey:
- system invariants  
- non‑mutation  
- non‑fusion  
- non‑inversion  

State cannot violate invariants.

## 5. State prevents semantic drift
State ensures:
- transitions do not imply meaning  
- transitions do not imply category  
- transitions do not imply identity  

State is a membrane.

## 6. State transitions are reversible
State transitions:
- do not mutate identity  
- do not mutate semantics  
- do not mutate structure  

State leaves no residue.

## 7. State cannot create new domains
State cannot:
- create new categories  
- create new semantics  
- create new containers  
- create new routing paths  

State only describes variation within what exists.

## 8. State must be explicit
State must be:
- declared  
- bounded  
- intentional  

Implicit state leads to drift.

## 9. State cannot interpret behavior
State does not:
- infer intent  
- interpret meaning  
- evaluate content  

State is structural, not cognitive.

## 10. State must remain isolated
State must not fuse with:
- identity  
- semantics  
- registry  
- placement  
- adjacency  
- routing  
- engine logic  

State is a standalone conceptual domain.
